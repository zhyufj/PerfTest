# Perftest - HTTP 性能测试工具

## 项目简介

这是一个使用 Go 语言开发的轻量级 HTTP 性能压测工具，用于接口性能测试与网络瓶颈分析。本项目为学习型项目，通过从头实现压测引擎，深入理解性能测试的核心原理与技术细节。

---

## 技术实现要点

| 技术模块 | 实现方案 |
|---------|---------|
| **并发模型** | 基于 Goroutine 的 M:N 调度模型，相比传统线程模型节省内存开销，1000 并发仅占用 < 100MB 内存 |
| **连接池策略** | 自定义 `http.Transport` 配置，支持 **连接复用** / **无连接池** 双模式，后者可模拟真实浏览器行为 |
| **统计算法** | 实时百分位统计（P50/P90/P95/P99），秒级采样存储历史数据用于趋势分析 |
| **流量控制** | Ramp-Up 渐进式加压机制，避免瞬时流量冲击造成的测试失真 |
| **跨平台构建** | 纯 Go 标准库实现，无第三方依赖，单文件编译，支持 Windows/Linux/macOS |

---

## 核心功能

✅ **多模式压测**
- 连接池复用模式：高性能服务器压测
- 无连接池模式：模拟浏览器行为，可用于测试路由器 NAT 性能瓶颈

✅ **渐进式加压**
- 支持 Ramp-Up 爬坡时间，线程数匀速增加
- 避免瞬时流量冲击，测试结果更接近真实场景

✅ **完整指标体系**
- 吞吐量（TPS）、成功率、错误率
- 响应时间统计：平均值、P50/P90/P95/P99 百分位、最大最小值
- 秒级性能趋势采集，支持性能拐点分析

✅ **可视化报告**
- 单文件 HTML 报告，无需依赖，直接打开
- 性能趋势双轴图表、响应时间百分位图
- 响应采样记录，便于错误定位与分析

✅ **友好交互体验**
- 交互式命令行，一步步配置参数
- 支持输入中途回退
- 测试过程随时终止，已测数据不丢失
- 配置文件导入导出

## 快速开始

### 方式一：交互式模式（推荐）

```bash
# Windows
perftest_v3.0.exe

# Linux/macOS
./perftest
```

### 方式二：快捷命令行模式

```bash
# 50 并发，压测 1 分钟
perftest.exe quick -c 50 -d 1m https://example.com/api

# 禁用连接池，模拟浏览器行为（测试路由器用）
perftest.exe quick -c 30 -d 2m -k n http://192.168.2.1
```

---

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-c` | 并发线程数 | 10 |
| `-d` | 压测时长（如 30s, 1m） | 30s |
| `-k` | 连接模式: y=复用, n=禁用 | y |
| `-m` | 请求方法 | GET |

---


## 编译运行

```bash
# 编译
go build -ldflags "-s -w" -o perftest.exe main.go

# 运行测试
go run main.go quick -c 5 -d 10s https://www.baidu.com
```
